Giám Sát Ca Học Discord

Tài liệu chi tiết hướng dẫn thiết lập, liên kết và vận hành hệ thống giám sát học tập tích hợp thời gian thực.

01

Bật Chế Độ Nhà Phát Triển (Developer Mode)

Để lấy được mã định danh tài khoản cá nhân, bạn cần bật công cụ nhà phát triển được tích hợp sẵn trên ứng dụng Discord:

  1. Mở ứng dụng Discord trên máy tính hoặc điện thoại.
  2. Nhấn vào biểu tượng bánh răng Cài đặt người dùng (User Settings) ở góc dưới cùng bên trái.
  3. Cuộn xuống danh mục bên trái, tìm và chọn mục Nâng cao (Advanced) nằm ở phần Cài đặt ứng dụng.
  4. Kích hoạt tùy chọn Chế độ dành cho nhà phát triển (Developer Mode).
02

Sao Chép ID Tài Khoản Discord

Sau khi đã kích hoạt chế độ nhà phát triển, bạn tiến hành lấy mã ID tài khoản của mình:

  1. Nhìn vào avatar hoặc tên tài khoản của bạn (ở góc dưới bên trái hoặc trong danh sách thành viên server).
  2. Click chuột phải (hoặc nhấn giữ trên điện thoại) vào tên/avatar của chính mình.
  3. Chọn dòng cuối cùng: Sao chép ID người dùng (Copy User ID).
  4. Bạn sẽ thu được một dãy số dài (Ví dụ: 318263590503694336). Đây chính là ID định danh Discord của bạn.
03

Liên Kết ID Lên Hệ Thống Web

Bước cuối cùng để hoàn tất việc kết nối tài khoản giúp hệ thống ghi nhận thời gian học voice tự động:

  1. Đăng nhập tài khoản học tập của bạn trên Website ExamHub.
  2. Truy cập vào trang Hồ sơ cá nhân (Profile) -> nhấn nút Chỉnh sửa (Edit).
  3. Tìm đến trường nhập Discord ID (nằm ngay dưới mục Số điện thoại).
  4. Dán dãy số ID đã sao chép ở Bước 2 vào và bấm Lưu thay đổi.
✨
Hoàn thành! Giờ đây mỗi khi bạn tham gia học voice trên Discord lớp học, hệ thống sẽ tự động ghi nhận thời gian và tích hoàn thành nhiệm vụ thời khóa biểu cho bạn.

01

Chạy SQL Database Migration

Thầy cần cấu hình cấu trúc cơ sở dữ liệu trên Supabase SQL Editor để hỗ trợ lưu trữ trạng thái học và lịch sử điểm danh Discord của học sinh.

💡
Mở Supabase Dashboard -> SQL Editor -> Tạo Query mới, dán đoạn mã bên dưới và nhấn Run:
migration-discord-monitor.sql
-- Bổ sung trường discord_id vào bảng profiles
ALTER TABLE public.profiles ADD COLUMN IF NOT EXISTS discord_id TEXT UNIQUE;

-- Điều chỉnh trạng thái hỗ trợ trong study_sessions
ALTER TABLE public.study_sessions DROP CONSTRAINT IF EXISTS study_sessions_status_check;
ALTER TABLE public.study_sessions ADD CONSTRAINT study_sessions_status_check 
    CHECK (status IN ('focusing', 'resting', 'offline', 'discord_class', 'discord_afk'));

-- Bổ sung trường đo lường thời lượng cuộc gọi
ALTER TABLE public.study_sessions ADD COLUMN IF NOT EXISTS discord_duration INTEGER DEFAULT 0;
ALTER TABLE public.study_sessions ADD COLUMN IF NOT EXISTS discord_deafened BOOLEAN DEFAULT false;
ALTER TABLE public.study_sessions ADD COLUMN IF NOT EXISTS discord_last_active TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now());
02

Liên Kết & Giám Sát Tài Khoản Học Sinh

Sau khi học sinh đã nhập Discord ID trên tài khoản của mình, thầy chỉ cần thực hiện kết nối trên giao diện quản trị để bắt đầu quan sát học sinh đó:

  1. Truy cập vào Đài Giám Sát Học Tập trên Web.
  2. Tại phần Kết nối học sinh, nhập chính xác Email đăng ký của học sinh rồi ấn Kết nối.
  3. Sau khi kết nối thành công, tên học sinh sẽ xuất hiện trong danh sách lựa chọn nhanh.
  4. Thầy chuyển sang tab Giám sát Discord để theo dõi trực quan biểu đồ thời gian học/treo máy và nhật ký chi tiết các cuộc gọi.
03

Cấu Hình Biến Môi Trường (Environment Variables)

Để đảm bảo luồng đồng bộ từ Discord Bot về Web Next.js được an toàn, thầy cần cấu hình các biến môi trường sau:

A. Cài đặt trên Vercel Dashboard (Cho Web):

  • SUPABASE_SERVICE_ROLE_KEY: Dùng để cập nhật trực tiếp checklist học tập của học sinh mà không bị chặn bởi bộ lọc chính sách bảo mật RLS.
  • DISCORD_SYNC_SECRET: Đặt mã token bảo mật bất kỳ (Ví dụ: discord_sync_secret_token_2026) dùng để kiểm soát quyền gọi Webhook.

B. Cấu hình file .env (Cho Discord Bot):

Tạo một tệp tin .env đặt trong thư mục chạy Bot Discord của thầy:

.env
DISCORD_BOT_TOKEN=TOKEN_BOT_DISCORD_CUA_BAN
CLASS_VOICE_CHANNEL_ID=ID_PHONG_THOAI_LOP_HOC_TREN_DISCORD
WEB_API_URL=https://examhub.vercel.app/api/study-sessions/discord-sync
DISCORD_SYNC_SECRET=discord_sync_secret_token_2026
04

Khởi Chạy Discord Bot

Mã nguồn bot Discord đã được chuẩn bị tại: scripts/discord-bot/tracker.js.

Các bước khởi chạy bot:

  1. Mở Terminal tại thư mục của dự án và cài đặt dependencies: 
    npm install discord.js axios dotenv
  2. Khởi động bot để bắt đầu lắng nghe sự kiện thoại: 
    node scripts/discord-bot/tracker.js

Lưu ý: Để hệ thống hoạt động 24/7 không gián đoạn, thầy nên triển khai chạy Bot này trên các máy chủ VPS (như Render, PM2 hoặc Docker).

05

Giám Sát Trực Quan Trên Đài Giám Sát

Khi học sinh tham gia học, thầy có thể giám sát toàn bộ tiến trình qua Web:

  • Trạng thái Realtime 🎧: Tự động đổi trạng thái sang Đang học qua Discord 🟢 màu xanh lá.
  • Phát hiện treo máy (AFK) 🚫: Nếu học sinh tắt tai nghe (Deafen) quá 10 phút, hệ thống lập tức ghi nhận trạng thái là discord_afk, nhấp nháy đỏ trên Đài Giám Sát của thầy và gửi thông báo nhắc nhở trực tiếp tới học sinh.
  • Thống kê thời lượng: Hệ thống vẽ biểu đồ cột thể hiện chính xác số phút học thực tế (xanh lá) và số phút treo máy (vàng cam) của từng ngày trong tuần.
