01

Chạy SQL Database Migration

Để hỗ trợ lưu trữ trạng thái học tập qua Discord của học sinh, thầy cần bổ sung cấu trúc cơ sở dữ liệu trên trang quản trị Supabase của mình.

⚠️
Thầy mở trang quản trị Supabase Dashboard -> chọn SQL Editor -> tạo một Query mới, sao chép nội dung tệp tin bên dưới và bấm Run.
migration-discord-monitor.sql
-- Bổ sung trường discord_id vào bảng profiles
ALTER TABLE public.profiles ADD COLUMN IF NOT EXISTS discord_id TEXT UNIQUE;

-- Điều chỉnh trạng thái hỗ trợ trong study_sessions
ALTER TABLE public.study_sessions DROP CONSTRAINT IF EXISTS study_sessions_status_check;
ALTER TABLE public.study_sessions ADD CONSTRAINT study_sessions_status_check 
    CHECK (status IN ('focusing', 'resting', 'offline', 'discord_class', 'discord_afk'));

-- Bổ sung trường đo lường thời lượng cuộc gọi
ALTER TABLE public.study_sessions ADD COLUMN IF NOT EXISTS discord_duration INTEGER DEFAULT 0;
ALTER TABLE public.study_sessions ADD COLUMN IF NOT EXISTS discord_deafened BOOLEAN DEFAULT false;
ALTER TABLE public.study_sessions ADD COLUMN IF NOT EXISTS discord_last_active TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now());
02

Liên Kết Tài Khoản Discord của Học Sinh

Để hệ thống nhận biết chính xác học sinh nào đang tham gia thoại Discord, thầy cần cập nhật tài khoản định danh Discord của học sinh vào cơ sở dữ liệu.

  1. Bật Developer Mode trên cài đặt Discord cá nhân của thầy.
  2. Click chuột phải vào tài khoản của học sinh trên Discord -> Chọn Copy User ID (Sẽ thu được dãy số có định dạng ví dụ như: 392182049182390281).
  3. Truy cập vào Supabase table editor -> Mở bảng profiles -> Tìm đúng dòng của học sinh -> Nhập dãy số ID vừa copy vào cột discord_id.
03

Cấu Hình Biến Môi Trường (Environment Variables)

Để đồng bộ an toàn từ xa, hệ thống Next.js Web và Discord Bot cần cấu hình các thông số bảo mật liên kết:

A. Cấu hình trên Vercel Dashboard (Dành cho Web):

  • SUPABASE_SERVICE_ROLE_KEY: Dùng để cập nhật trực tiếp checklist học tập của học sinh mà không bị chặn bởi bộ lọc chính sách bảo mật RLS.
  • DISCORD_SYNC_SECRET: Đặt mã token bảo mật bất kỳ (Ví dụ: super_secret_discord_desk_key) dùng để kiểm soát quyền gọi Webhook.

B. Cấu hình file .env (Dành cho Discord Bot):

Tạo một tệp tin .env đặt trong thư mục chạy Bot Discord của thầy:

.env
DISCORD_BOT_TOKEN=TOKEN_BOT_DISCORD_CUA_BAN
CLASS_VOICE_CHANNEL_ID=ID_PHONG_THOAI_LOP_HOC_TREN_DISCORD
WEB_API_URL=https://examhub.vercel.app/api/study-sessions/discord-sync
DISCORD_SYNC_SECRET=super_secret_discord_desk_key
04

Khởi Chạy Discord Bot

Mã nguồn bot Discord đã được viết sẵn tại đường dẫn thư mục: scripts/discord-bot/tracker.js.

Để khởi chạy bot:

  1. Di chuyển vào thư mục bot và cài đặt các thư viện cần thiết: 
    npm install discord.js axios dotenv
  2. Khởi động bot Discord lên để lắng nghe: 
    node tracker.js

Lưu ý: Thầy có thể triển khai chạy bot này trên máy tính cá nhân trong suốt buổi học hoặc lưu trữ trực tiếp trên máy chủ VPS để hệ thống hoạt động 24/7.

05

Theo Dõi Trực Quan Trên Đài Giám Sát

Khi các bước cấu hình hoàn tất, thầy chỉ cần truy cập vào Đài Giám Sát Học Tập trên Web để xem kết quả:

  • Theo dõi Realtime: Trạng thái của học sinh tự động đổi sang Đang học qua Discord 🎧 màu xanh khi bắt đầu vào phòng học.
  • Cảnh báo AFK 🚫: Nếu học sinh tắt tai nghe hoặc tắt tiếng (Mute/Deafen) để treo máy, giao diện lập tức nhấp nháy dòng chữ màu cam Treo máy/Tắt tiếng Discord 🚫 và phát âm thanh/hiển thị cảnh báo trực quan cho giáo viên biết.
  • Đo lường thời gian thực tế: Thanh tiến độ YPT-style sẽ hiển thị rõ ràng số phút học tích lũy (ví dụ: Đã học: 45 / 130 phút).
  • Tự động duyệt Checklist: Khi số phút vượt ngưỡng 130 phút, một checklist nhiệm vụ trên trang cá nhân của học sinh sẽ tự động được đánh dấu hoàn thành 100% thời lượng bài học.
